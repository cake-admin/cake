One Lenovo "Cake" Design System

The One Lenovo “Cake” Design System is a design framework aimed at creating a unified design language across all Lenovo software. Its purpose is to ensure consistency, improve usability, and enhance the overall user experience of Lenovo’s diverse range of digital products.

Why use Cake?

Teams using the design system will benefit from faster design and development due to pre-designed components, built-in accessibility that meets WCAG 2.2 standards, and modern interface design that aligns with the latest trends and best practices, resulting in more cohesive, accessible, and efficient software.

Three pillars:

Cake design system is built upon three foundational pillars— Accessibility, Brand, and System, each playing a vital role in achieving a seamless, recognizable, and high-quality interface for Lenovo users worldwide.

Accessibility

Each component meets WCAG 2.2 AA accessibility standards, delivering an inclusive UX that meets diverse needs.

Brand

Utilizes Lenovo brand colors and visual identity for a cohesive, recognizable look across products.

System

A systemized approach streamlines the creation of new features and products, ensuring uniformity across interfaces.

Resources

Cake - Figma

Core libraries that contain product agnostic components used as foundational building blocks for all software at Lenovo.

Subsystems - Figma

Libraries that contain product or theme-specific components built from the core library. These exist to provide unique or new components that may not be availible in Cake.
